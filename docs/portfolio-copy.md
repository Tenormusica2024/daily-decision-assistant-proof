# Portfolio Copy

## Short label

AI Secretary Daily Decision Proof

## One-line description

Sample-first AI secretary proof that turns mixed daily signals into a focus/defer/no-go decision brief and queues external actions for human confirmation.

## Portfolio card copy

A daily decision proof for safe AI secretary workflows. It evaluates mixed signals such as deadlines, meetings, blocked tasks, and proof-asset work, then creates a small decision brief with today focus, defer, and no-go groups. The demo uses synthetic fixtures and requires no external API, so it can be reviewed without exposing private operational data.

## B2B proposal angle

This pattern is useful when a business wants AI to help decide what should be handled today, but does not want AI to mutate calendars, send messages, or update task systems automatically. The key value is the safety boundary: AI can rank, explain, and queue actions while humans approve external changes.

## Interview explanation

This is not a full task manager. I built it as a proof of an agent operations pattern: separate decision support from execution. The assistant can prioritize work and explain why, but any external action is placed into a confirmation queue first.

## Do not overclaim

Avoid saying:

- "Autonomous project manager"
- "Automatically changes schedules"
- "Works on real calendar/email by default"

Prefer saying:

- "Sample-first proof"
- "Decision support"
- "Human confirmation before action"
- "Public-safe synthetic fixtures"
